Merlin:
- bestehende Instanzen von ECORE-Modellen können nicht importiert werden:
	- kein Layout-Algorithmus
	- keine Möglichkeit bspw. aus ECORE-Editor Instanzen auf GUI zu ziehen
	--> kein richtiges Refactoring
- Verwendung eigener Connections (z. B. mit Pfeil) - Shapes nicht ohne Weiteres möglich. Zumeist: Event loop exception
- keine eigenen Shapes für Komponenten: eigene Label schwer zu realisieren. Bei Verwendung eigener Implementierungen, Anpassung oder vorgegebener Implementierungen aus GEF: Event Loop Exceptions
- Änderungen an GEF GEN MODEL nach Reimport von EMF-Modellen sind erneut von Hand einzupflegen --> beinhaltet u. a. Shapes-Anpassung, Connection zu "Real Connection"

- Komponentenmodell: Nur Protokoll auf einzelner Schnittstelle, nicht auf Menge von Schnittstellen möglich.

ECORE:
- Aus RSA Export von ECORE und Import in EMF: Zwar XML:
		<xsd:annotation>
		   <xsd:appinfo source="http://www.eclipse.org/emf/2002/Ecore" 
		   ecore:key="constraints">
			 NonNegativeQuantity ValidShipDate
		   </xsd:appinfo>
		</xsd:annotation>
	damit Codegerüst für Validitätsprüfung angelegt. Jedoch Codegerüst auf "if(false)" gesetzt. Keine Constraints im ECORE-Modell angelegt.
	--> Abhilfe: Export aus RSA zu *UML2*. Darin korrekt Constraints-Body (OCL) angelegt. Bei Import in EMF ebenfalls Constraints in ECORE-Modell vorhanden.
- EMF kann keine OCL-Constraints (als nicht-Java-Quellcode) prüfen. --> Kent OCL Library
  

RSA:
