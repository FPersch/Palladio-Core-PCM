CodeSmith Frequently Asked Questions

 

 

Q: How can I add comments to my template?

 

A:  You need to include your comments inside the <script> or <% %> tags and then either use the comment character of the language you are using or the CodeSmith comment character For example:

 

<%-- Comments --%> - CodeSmith

<%’ Comments %> - VB.NET

<% // Comments %> - C#

<% /* Comments */ %> - C#

 

 

 

Q: How can I create a property that has a dropdown of values to choose from?

 

A: You can define an enumeration and then set the type of your property to that enumeration, here is an example:

 

<%@ Property Name=”CollectionType” Type=”CollectionTypeEnum” Category=”Collection” Description=”Type of collection” %>

 

<script runat=”tempate”>

public enum CollectionTypeEnum

{

  Vector,

  HashTable,

  SortedList

}

</script>

 

 

 

Q: I am generating ASP.NET code from my templates and the ASP.NET tags are causing CodeSmith to get confused.

 

A: Change the ASP.NET tags to <%% instead of <%, this will escape the tag and allow it to be passed down to the output of the template instead of being processed as template code.

 

 

Q: How can I declare a constant in my template?

 

A: Constants must be declared inside <script runat=”template”> tags.

 

Example:

 

<script runat=”template”>

private const string MY_CONST = “example”;

</script>

 

 

Q: How can I debug my template?

 

A: To debug a template it needs to be compiled in debug mode, to do this add debug=true to the CodeTemplate directive. You can then set break points using Debugger.Break() in your code.

 

Example:

 

<%@ CodeTemplate Language="C#" TargetLanguage="T-SQL"

      Description="Debugging your template" Debug="true" %>

     

<% Debugger.Break(); %>

 

 

 

Q: How can I add a property that lets me select a folder?

 

A: Using the Editor attribute you can create a property that allows you to select a folder when running the template.

 

Example:

 

[Editor(typeof(System.Windows.Forms.Design.FolderNameEditor), typeof(System.Drawing.Design.UITypeEditor))]

public string OutputDirectory

{

      get {return _outputDirectory;}

      set {_outputDirectory= value;}

}

 

 

 

Q: How can I use sub-templates?

 

A: Sub templates can be used to call another template from inside of your template by accessing the CodeSmith API. See below for an example of how to do this.

 

Example:

 

<%

foreach (TableSchema table in SourceDatabase.Tables)

{

   OutputSubTemplate(table);

}

%>

 

<script runat="template">

private CodeTemplate _mySubTemplate;

 

 

 

 

[Browsable(false)]

public CodeTemplate MySubTemplate

{

   get

   {

      if (_mySubTemplate == null)

      {

         CodeTemplateCompiler compiler = new CodeTemplateCompiler(this.CodeTemplateInfo.DirectoryName + "MySubTemplate.cst");

         compiler.Compile();

        

         if (compiler.Errors.Count == 0)

         {

            _mySubTemplate = compiler.CreateInstance();

         }

         else

         {

            for (int i = 0; i < compiler.Errors.Count; i++)

            {

               Response.WriteLine(compiler.Errors[ i].ToString());

            }

         }

      }

     

      return _mySubTemplate;

   }

}

 

public void OutputSubTemplate(TableSchema table)

{

   MySubTemplate.SetProperty("SourceTable", table);

   MySubTemplate.SetProperty("IncludeDrop", false);

   MySubTemplate.SetProperty("InsertPrefix", "Insert");

  

   MySubTemplate.Render(Response);

}

</script>

 

Template Download: SubTemplatesExample.cst, and MySubTemplate.cst

 

 

 

Q: What assemblies and namespaces are loaded by default into a template?

 

A:

 

Assemblies: mscorlib, System, System.Xml, System.Data, System.Drawing, Microsoft.VisualBasic, System.Windows.Forms, CodeSmith.Engine

 

Namespaces: System, System.Data, System.Diagnostics, System.ComponentModel, Microsoft.VisualBasic, CodeSmith.Engine

 

SchemaExplorer

 

Q: Is it possible to determine if a field is an Identity field using the SchemaExplorer?

 

A: The extended properties collection of the column contains a property called "CS_IsIdentity". If this is set to true then the column is an identity column.

 

Example:

 

Identity Field = <% foreach(ColumnSchema cs in SourceTable.Columns) { 

      if( ((bool)cs.ExtendedProperties["CS_IsIdentity"].Value) == true)

      {

            Response.Write(cs.Name);

      }

}

%>

 

Template Download: CS_Identity_Example.cst

 

 

 

Q: How can I detect input and output parameters of a stored procedure using the SchemaExplorer?

 

A: The CommandSchema object contains both input and output parameter collections which can be used to read these parameters.

 

Example:

 

Input Parameters:

<%foreach(ParameterSchema ps in SourceProcedure.AllInputParameters)

{

      Response.Write(ps.Name);

      Response.Write("\n");

}

%>

 

 

Output Parameters:

<%foreach(ParameterSchema ps in SourceProcedure.AllOutputParameters)

{

      Response.Write(ps.Name);

      Response.Write("\n");

}

%>

 

Template Download: InputOutputParameterExample.cst
