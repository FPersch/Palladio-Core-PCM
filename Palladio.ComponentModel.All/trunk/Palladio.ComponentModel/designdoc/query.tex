\section{Suchanfragen an das Modell}

Die Suche von Entitäten und die Anfrage von Beziehungen zwischen diesen ist eine der wichtigsten Vorraussetzungen zur sinnvollen Nutzung des Komponentenmodells. Aufbauend auf diesen lassen spezielle  Abfragemethoden in Form von Vergleichs- oder Navigationsoperationen  entwickeln.

Dieses Kapitel stellt die bereits entworfene Basisfunktionalität zur Bearbeitung von Anfragen an das Komponentenmodell vor. Es folgt anschließend die Präsentation zweier in Planung befindlicher spezieller Anfragemethoden.

\subsection{Allgemeine Anfragen}
\label{sec:query:allgemein}

Allgemeine Anfragen beschränken sich auf die Eigenschaften von und die Beziehungen zwischen den Entitäten. Erstere sind hierbei in die drei Ebenen \emph{Type-Level}, \emph{Im\-ple\-men\-ta\-tion-Level} und \emph{Deployment-Level} (vgl. \cite{lit:cm}) unterteilt.

Alle Anfragemethoden sind über das Interfaces \verb+IQuery+ direkt aus der das gesamte Modell kapselnden Klasse \verb+ComponentModelEnvironment+ erreichbar. Dieses Interface er\-mög\-licht gezielte Erfragung von Beziehungen zwischen Entitäten pro Ebene. Die Entitäten selber und damit auch ihre Attribute sind unter Verwendung der im Interface \verb+IQueryRepository+ definierten Methoden erreichbar. Hierbei ist der entsprechenden Methode die ID der Entität zu übergeben. Kann die Entität nicht gefunden werden, so wird eine entsprechende Ausnahme ausgelöst.

Beziehungen zwischen den Entitäten müssen ähnlich wie bei den Buildern (vgl. Kapitel \ref{sec:builder}) ausgehendend von einem Bezugspunkt gestellt werden. Besteht beispielsweise Interesse an allen Schnittstellen einer Komponente, so ist unter Verwendung der Methode \verb+QueryComponent+ die entsprechende Komponente zu befragen. Als Ergebnis liefern die Methoden je nach Art entweder eine einzige oder eine Liste von IDs zurück. Gemäß dem Prinzip der Trennung von Entitäten und deren IDs (vgl. Kapitel \ref{sec:kern:beschr:ids}) werden nie die Entitäten selber als Parameter verlangt oder als Ergebnis zurückgegeben.

Aufbauend hierauf lassen sich Anfragen auf höherer Abstraktionsebene stellen. Hierzu gehört beispielsweise die Navigation im Modell oder der Vergleich von Entitäten anhand verschiedener Kriterien. Hintergründe, Anforderungen und Lösungsideen dieser beiden in Planung befindlichen Bestandteile werden in den zwei folgenden Teilkapiteln vorgestellt.
